# Workshop 3: การสร้างการแสดงผล (Visualization)

## สถานการณ์จำลอง

ผู้บริหารต้องการเห็นข้อมูลงบประมาณและการเบิกจ่ายในรูปแบบกราฟและตารางที่ตอบคำถามทางธุรกิจได้อย่างรวดเร็ว

## วัตถุประสงค์

1. ออกแบบหน้ารายงานด้วยองค์ประกอบพื้นฐาน (ชื่อรายงาน กราฟ ตาราง)
2. สร้าง Visualization หลักเพื่อช่วยตอบคำถามเรื่องแหล่งงบประมาณ การเบิกจ่าย และรายละเอียดตามหน่วยปฏิบัติ

## การตั้งชื่อรายงาน

ใช้ Text ใส่หัวข้อใหญ่ด้านบนของหน้า:  
`รายงานติดตามงบประมาณ ประจำปี 2567`

## กราฟสัดส่วนแหล่งงบประมาณ (Pie Chart)

- Dimension: แหล่งงบประมาณ  
- Metric: งบประมาณ  
- คำถามที่ตอบ: งบปีนี้มาจากส่วนกลางหรือภูมิภาคมากกว่ากัน?

## สถานะการเบิกจ่ายภาพรวม (Gauge/Scorecard)

1. สร้าง Field ใหม่ชื่อ **% การเบิกจ่าย** ด้วยสูตร  
   `SUM(ยอดเบิกจ่าย) / SUM(งบประมาณ)`  
2. ใช้ Field นี้เป็น Metric และตั้ง Type เป็น Percent (%)  
3. ใช้เพื่อตอบคำถาม: เบิกจ่ายไปแล้วกี่เปอร์เซ็นต์?

## การเบิกจ่ายจำแนกตามกลุ่ม (Stacked Bar / Column)

- กราฟ 4.1 (แหล่งงบประมาณ): Dimension = แหล่งงบประมาณ, Metric = ยอดเบิกจ่าย + งบประมาณคงเหลือ  
- กราฟ 4.2 (ชื่อกิจกรรม): Dimension = ชื่อกิจกรรม, Metric = ยอดเบิกจ่าย + งบประมาณคงเหลือ  
- กราฟ 4.3 (หน่วยปฏิบัติ - แนวตั้ง): Dimension = หน่วยปฏิบัติ, Metric = ยอดเบิกจ่าย + งบประมาณคงเหลือ

## ตารางสรุปรายละเอียด (Pivot Table)

- Row Dimension: หน่วยปฏิบัติ  
- Column Dimension: ชื่อกิจกรรม  
- Metrics: งบประมาณ, ยอดเบิกจ่าย, งบประมาณคงเหลือ (GF)
